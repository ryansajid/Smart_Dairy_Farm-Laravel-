## 1. INTRODUCTION

### 1.1 Purpose

The Sprint Retrospective is a Scrum ceremony held at the end of each sprint to:
- Reflect on the sprint that just concluded
- Identify what went well, what could improve, and action items
- Create a plan for implementing improvements
- Foster continuous improvement in the team

### 1.2 Principles

- **Safe Environment**: All feedback is welcomed without judgment
- **Everyone Participates**: Every team member has a voice
- **Focus on System, Not People**: Blameless culture
- **Actionable Outcomes**: Concrete improvement actions
- **Timeboxed**: Typically 60-90 minutes for 2-week sprint

### 1.3 When and Duration

| Sprint Length | Retrospective Duration |
|---------------|------------------------|
| 1 week | 45 minutes |
| 2 weeks | 60-90 minutes |
| 3 weeks | 90-120 minutes |
| 4 weeks | 120-150 minutes |

**Schedule**: Friday afternoon, after Sprint Review

## 2. RETROSPECTIVE FORMATS

### 2.1 Start-Stop-Continue

**Best for**: Quick retrospectives, new teams

**Format**:
| Category | Questions |
|----------|-----------|
| **Start** | What should we start doing? |
| **Stop** | What should we stop doing? |
| **Continue** | What should we continue doing? |

**Time**: 60 minutes

### 2.2 4Ls (Liked, Learned, Lacked, Longed For)

**Best for**: Comprehensive feedback, balanced perspective

**Format**:
| Category | Questions |
|----------|-----------|
| **Liked** | What did you like about this sprint? |
| **Learned** | What did you learn? |
| **Lacked** | What was missing? |
| **Longed For** | What did you wish for? |

**Time**: 75 minutes

### 2.3 Sailboat

**Best for**: Visual representation, goal-oriented teams

**Elements**:
- **Wind**: What pushed us forward? (Positive forces)
- **Anchors**: What held us back? (Impediments)
- **Rocks**: What risks do we see ahead?
- **Island**: What's our goal/destination?

**Time**: 90 minutes

### 2.4 Mad-Sad-Glad

**Best for**: Emotional check-in, team morale focus

**Format**:
| Category | Focus |
|----------|-------|
| **Mad** | What frustrated you? |
| **Sad** | What disappointed you? |
| **Glad** | What made you happy? |

**Time**: 60 minutes

### 2.5 Timeline

**Best for**: Multi-sprint reviews, release retrospectives

**Format**:
- Draw timeline of sprint/release
- Plot events (positive above, negative below)
- Discuss patterns and insights

**Time**: 90-120 minutes

### 2.6 STAR (Strengths, Threats, Actions, Results)

**Best for**: SWOT-style analysis, strategic focus

**Format**:
| Category | Questions |
|----------|-----------|
| **Strengths** | What are we good at? |
| **Threats** | What could harm us? |
| **Actions** | What will we do? |
| **Results** | What outcomes do we expect? |

**Time**: 75 minutes

### 3.2 Detailed Phase Activities

#### Phase 1: Set the Stage (5 minutes)

**Activities**:
- Welcome everyone
- State the purpose
- Review working agreements
- Optional: Safety check (anonymous voting)

**Safety Check Questions**:
- "On a scale of 1-5, how safe do you feel sharing honestly?"
- "Are there any topics that are off-limits?"

#### Phase 2: Gather Data (15 minutes)

**Individual Reflection (5 min)**:
- Each person writes on sticky notes (physical or digital)
- One idea per note
- Silent activity

**Group Sharing (10 min)**:
- Each person shares their notes
- Place on board/tool
- No discussion yet

#### Phase 3: Generate Insights (15 minutes)

**Grouping**:
- Cluster similar items together
- Identify themes

**Dot Voting** (if needed):
- Each person gets 3-5 dots
- Vote on most important items
- Focus discussion on top-voted items

#### Phase 4: Decide Actions (15 minutes)

**Create Action Items**:
- Focus on top 2-3 improvement areas
- Make actions SMART:
  - **S**pecific
  - **M**easurable
  - **A**chievable
  - **R**elevant
  - **T**ime-bound

**Assign Ownership**:
- Each action has an owner
- Set due dates
- Add to next sprint backlog if needed

#### Phase 5: Close (10 minutes)

**Summarize**:
- Recap action items
- Confirm owners and dates

**Appreciations**:
- Team members appreciate each other
- Shout-outs for great work

**Retro of the Retro**:
- "What worked well in this retrospective?"
- "What could we improve next time?"

## 4. ACTION ITEMS

### 4.1 Action Item Format

| Field | Description | Example |
|-------|-------------|---------|
| **ID** | Unique identifier | RETRO-001 |
| **Action** | What to do | "Reduce code review time" |
| **Owner** | Who is responsible | "Tech Lead" |
| **Due Date** | When to complete | "Next Sprint" |
| **Status** | Current state | "Open" |
| **Notes** | Additional details | "Implement PR templates" |

### 4.2 Action Item Categories

| Category | Examples |
|----------|----------|
| **Process** | "Improve daily standup efficiency" |
| **Technical** | "Increase test coverage to 80%" |
| **Communication** | "Add mid-sprint check-in" |
| **Tools** | "Set up automated deployments" |
| **Team** | "Organize team building activity" |

### 4.3 SMART Action Items

**Bad**: "Improve communication"
**Good**: "Add 15-minute mid-sprint check-in every Wednesday at 2 PM, starting next sprint, owned by Scrum Master"

## 5. REMOTE RETROSPECTIVES

### 5.1 Tools

| Tool | Purpose | Best For |
|------|---------|----------|
| **Miro** | Collaborative board | Visual formats |
| **Mural** | Digital whiteboard | Large teams |
| **FunRetro** | Purpose-built retro tool | Quick setup |
| **Parabol** | Async + sync | Distributed teams |
| **Trello** | Kanban board | Simple formats |

### 5.2 Remote Best Practices

**Before**:
- Test tools beforehand
- Send board link in advance
- Have backup communication channel

**During**:
- Use video (cameras on)
- Use timer for activities
- Call on quieter members
- Use breakout rooms for large teams

**After**:
- Share board screenshot
- Export action items
- Schedule follow-ups

### 5.3 Time Adjustments for Remote

| Activity | In-Person | Remote |
|----------|-----------|--------|
| Set the Stage | 5 min | 5 min |
| Gather Data | 15 min | 20 min |
| Generate Insights | 15 min | 20 min |
| Decide Actions | 15 min | 20 min |
| Close | 10 min | 10 min |
| **Total** | **60 min** | **75 min** |

## 6. FOLLOW-UP

### 6.1 Action Item Tracking

**Methods**:
- Add to Sprint Backlog as improvement items
- Track in separate action item log
- Review at next retrospective

### 6.2 Follow-up Schedule

| Timing | Activity |
|--------|----------|
| **Daily** | Quick check on action progress during standup |
| **Weekly** | Review with team mid-sprint |
| **Next Retro** | Review completion of previous actions |

### 6.3 Escalation

If action items are not completed:
1. Discuss in next retrospective
2. Identify blockers
3. Reassign if needed
4. Escalate to management if systemic

## 7. SAFETY CHECK

### 7.1 Anonymous Safety Check

**Scale**: 1 (Not safe) to 5 (Very safe)

**Interpretation**:
| Average | Meaning | Action |
|---------|---------|--------|
| 4.5 - 5.0 | Very safe | Proceed normally |
| 3.5 - 4.4 | Mostly safe | Monitor, encourage sharing |
| 2.5 - 3.4 | Concerns | Address safety concerns first |
| 1.0 - 2.4 | Unsafe | Stop retro, address root cause |

### 7.2 Creating Psychological Safety

**Do**:
- Start with appreciations
- Model vulnerability (Scrum Master shares first)
- Thank people for honest feedback
- Focus on process, not people

**Don't**:
- Allow blame or judgment
- Dismiss concerns
- Force participation
- Rush through uncomfortable topics

## 9. ANTI-PATTERNS

### 9.1 Common Anti-Patterns

| Anti-Pattern | Description | Solution |
|--------------|-------------|----------|
| **Blame Game** | Finger-pointing at individuals | Focus on system, not people |
| **Complaint Session** | Only problems, no solutions | Require action items |
| **Silent Room** | No one participates | Use structured activities |
| **Solution Jumping** | Skip data gathering | Follow the phases |
| **Action Overload** | Too many action items | Limit to 2-3 items |
| **Same Actions** | Same items every sprint | Review previous actions |
| **Manager Speaking** | Manager dominates | Manager listens only |
| **Skipping Retro** | "We're too busy" | Make it non-negotiable |

### 9.2 Warning Signs

- Actions never completed
- Same complaints every sprint
- Decreasing participation
- Negative atmosphere
- People skipping retrospectives
