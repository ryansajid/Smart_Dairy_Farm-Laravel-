## 1. Introduction

### 1.1 Purpose

This document defines the standards, processes, and resources for creating video tutorial content for the Smart Dairy Web Portal System. Video tutorials provide an engaging, visual learning experience that complements written documentation and accelerates user adoption.

### 1.2 Video Strategy

The Smart Dairy video tutorial strategy follows these principles:

- **Microlearning**: Short, focused videos for quick consumption
- **Progressive Learning**: Structured curriculum from basic to advanced
- **Multilingual**: Primary content in Bengali with English subtitles
- **Accessibility**: Captions, transcripts, and mobile-friendly formats
- **Searchability**: Organized tagging and categorization for easy discovery

### 1.3 Target Audience

| Audience | Content Focus | Preferred Format |
|----------|---------------|------------------|
| Farm Owners | Management dashboards, reports, analytics | Process walkthroughs |
| Farm Managers | Daily operations, team management | Feature demos |
| Field Officers | Mobile app usage, data entry | Quick tips |
| Sales Staff | Order processing, customer management | Complete tutorials |
| Warehouse Staff | Inventory, dispatch, receiving | Process walkthroughs |
| Accountants | Financial reports, transactions | Complete tutorials |
| Administrators | System setup, user management | Process walkthroughs |

### 1.4 Video Formats

| Format | Duration | Resolution | Frame Rate | Audio |
|--------|----------|------------|------------|-------|
| Web | Standard | 1920x1080 | 30fps | Stereo 48kHz |
| Mobile | Vertical | 1080x1920 | 30fps | Stereo 48kHz |
| Download | High | 2560x1440 | 30fps | Stereo 48kHz |

### 1.5 Distribution Channels

- **YouTube**: Primary hosting (unlisted playlists)
- **Internal LMS**: Learning Management System integration
- **Mobile App**: Embedded video player for on-the-go learning
- **Help Center**: Embedded videos in knowledge base articles

## 3. Video Production Standards

### 3.1 Technical Specifications

#### Resolution Requirements

| Platform | Minimum | Recommended | Maximum |
|----------|---------|-------------|---------|
| Desktop | 1920x1080 | 1920x1080 | 2560x1440 |
| Mobile | 1080x1920 | 1080x1920 | 1080x1920 |
| Tablet | 1920x1080 | 1920x1080 | 2560x1440 |

**Aspect Ratios**:
- Standard: 16:9 (landscape)
- Mobile: 9:16 (portrait)
- Square: 1:1 (social media)

#### Frame Rate & Encoding

- **Frame Rate**: 30fps constant
- **Codec**: H.264 (AVC) or H.265 (HEVC)
- **Container**: MP4
- **Bitrate**:
  - 1080p: 8-12 Mbps
  - 1440p: 16-24 Mbps

### 3.2 Audio Quality Standards

#### Recording Specifications

| Parameter | Requirement |
|-----------|-------------|
| Sample Rate | 48 kHz |
| Bit Depth | 24-bit |
| Channels | Stereo (2.0) |
| Format | WAV (source), AAC (delivery) |
| Bitrate | 192 kbps minimum |

#### Audio Processing

1. **Noise Reduction**: Remove background noise below -60dB
2. **Normalization**: Peak at -3dB, RMS at -16dB (voice), -20dB (music)
3. **Compression**: 3:1 ratio, -18dB threshold
4. **EQ**: High-pass filter at 80Hz, slight boost at 3-5kHz for clarity

#### Music Guidelines

- Royalty-free background music only
- Volume level: 20-30% of voice volume
- Duration: Continuous loop, fading during voiceover
- Style: Professional, upbeat, culturally appropriate

### 3.3 Screen Recording Software

#### Recommended Tools

| Software | Platform | Best For | License |
|----------|----------|----------|---------|
| Camtasia | Windows/Mac | Professional editing | Commercial |
| OBS Studio | All | Free, customizable | Open Source |
| ScreenFlow | Mac | Mac-native workflow | Commercial |
| Loom | Web | Quick recording | Freemium |
| Snagit | Windows/Mac | Simple captures | Commercial |

#### Recording Settings

```yaml
Capture Settings:
  Frame Rate: 30fps
  Encoder: x264 (software) or NVENC (hardware)
  Color Space: sRGB
  Mouse Capture: Enabled with highlighting
  
Cursor Settings:
  Highlight: Yellow circle
  Click Animation: Ripple effect
  Size: 1.5x normal
```

### 3.4 Bengali Voiceover Guidelines

#### Language Standards

- **Dialect**: Standard Bengali (Shadhu-bhasha for formal, Cholito-bhasha for casual)
- **Accent**: Neutral, easily understood across Bangladesh
- **Script**: Written in Bengali script
- **Transliteration**: Provide English phonetic guide for technical terms

#### Voice Talent Requirements

| Attribute | Specification |
|-----------|---------------|
| Gender | Male or Female (diversity in library) |
| Age Range | 25-45 years |
| Tone | Professional, friendly, clear |
| Pace | 120-140 words per minute |
| Clarity | Articulate, minimal regional accent |

### 6.3 Recording Guidelines

#### Pre-Recording Checklist

**Environment Preparation**:
- [ ] Browser cache cleared
- [ ] Test data loaded (no production data)
- [ ] Screen resolution set to 1920x1080
- [ ] Notifications disabled
- [ ] Desktop cleaned (no personal files visible)
- [ ] Browser bookmarks bar hidden
- [ ] URL bar shows test domain only

**Audio Preparation**:
- [ ] Quiet recording environment
- [ ] Microphone tested
- [ ] Pop filter in place
- [ ] Script printed or displayed on separate device
- [ ] Water available

**Recording Session**:
- [ ] Record in segments (3-5 minutes each)
- [ ] Pause 3 seconds at start and end of each segment
- [ ] Speak clearly and at consistent pace
- [ ] Mark mistakes with clap or verbal marker
- [ ] Record multiple takes for critical sections

#### Recording Do's and Don'ts

| âœ… DO | âŒ DON'T |
|-------|----------|
| Use consistent cursor highlighting | Move cursor rapidly or unnecessarily |
| Zoom in on small interface elements | Show sensitive customer data |
| Pause briefly between steps | Rush through important details |
| Use keyboard shortcuts visibly | Click randomly without purpose |
| Show loading states in real-time | Edit out all waiting time |
| Highlight the area of focus | Assume viewers know where to look |

### 6.4 Editing Standards

#### Editing Workflow

1. **Rough Cut** (Day 1)
   - Import all footage
   - Arrange in sequence
   - Sync audio with video
   - Cut obvious mistakes

2. **Fine Cut** (Day 2)
   - Tighten transitions
   - Add B-roll where needed
   - Insert zooms and highlights
   - Add callout graphics

3. **Graphics Pass** (Day 3)
   - Add intro and outro
   - Create lower thirds
   - Add progress indicators
   - Insert chapter markers

4. **Audio Mix** (Day 4)
   - Balance levels
   - Add background music
   - Apply noise reduction
   - Add sound effects

5. **Export and QC** (Day 5)
   - Export multiple formats
   - Review full video
   - Check captions sync
   - Verify all links work

#### Annotation Standards

| Element | Style | Duration | Placement |
|---------|-------|----------|-----------|
| Click Highlight | Yellow circle, pulse | 1 second | Around cursor |
| Text Callout | White text, black outline | 3-5 seconds | Near relevant element |
| Arrow | Red, animated | 2-3 seconds | Pointing to element |
| Box Highlight | Blue border, semi-transparent fill | As needed | Around area |
| Step Number | Circle with number | 5 seconds | Top-left corner |
| Keyboard Shortcut | Key icon + text | 2 seconds | Bottom-center |

### 9.3 Archive Policy

#### Archiving Criteria

**Videos are archived when**:
- Feature is deprecated
- UI completely redesigned
- Process no longer exists
- Content merged into other videos
- Version 3.0+ released (keep v2.x for reference)

#### Archive Process

1. **Pre-Archive Checklist**:
   - [ ] Verify replacement video exists
   - [ ] Update all links pointing to video
   - [ ] Add redirect notice on old URL
   - [ ] Document reason for archiving
   - [ ] Notify users who recently viewed

2. **Archive Actions**:
   - Move video to Archive playlist
   - Add [ARCHIVED] prefix to title
   - Update description with replacement link
   - Keep unlisted (don't delete)
   - Update LMS to point to new video

3. **Post-Archive**:
   - Add to archive index
   - Update documentation
   - Archive source files
   - Remove from active learning paths

#### F.2 Recommended Configuration

**For Training Teams (Recommended)**:
- Primary: Camtasia Studio
  - Purpose: Screen recording and editing
  - Pros: All-in-one, easy learning curve, good annotations
  - Cons: Limited advanced features
  
- Secondary: Adobe Premiere Pro
  - Purpose: Complex projects, advanced effects
  - Pros: Industry standard, extensive features
  - Cons: Expensive subscription, steep learning curve

**For Budget-Conscious Teams**:
- Recording: OBS Studio (Free)
- Editing: DaVinci Resolve (Free version)
- Graphics: Canva (Free tier)
