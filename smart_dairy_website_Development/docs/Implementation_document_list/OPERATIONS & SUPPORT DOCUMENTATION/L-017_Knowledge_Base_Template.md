## 1. Introduction

### 1.1 Purpose

This document establishes the standards, templates, and guidelines for creating, managing, and maintaining the Smart Dairy Ltd. Knowledge Base (KB). It serves as the authoritative reference for all knowledge management activities within the organization.

### 1.2 Knowledge Base Purpose

The Smart Dairy Knowledge Base is a centralized repository of organizational knowledge designed to:

| Objective | Description |
|-----------|-------------|
| **Enable Self-Service** | Empower users to find solutions independently, reducing support ticket volume |
| **Preserve Knowledge** | Capture and retain institutional knowledge, reducing dependency on individual experts |
| **Ensure Consistency** | Provide standardized, accurate information across all support channels |
| **Accelerate Onboarding** | Reduce training time for new employees through comprehensive documentation |
| **Improve Efficiency** | Enable faster issue resolution by providing readily accessible information |
| **Support Decision Making** | Provide data-driven insights through analytics and user feedback |

## 3. Article Types

### 3.1 Article Type Overview

| Type | Purpose | Average Length | Target Resolution Time |
|------|---------|----------------|------------------------|
| **FAQ** | Quick answers to common questions | 100-300 words | 2 minutes |
| **How-To** | Step-by-step procedural guidance | 300-800 words | 5-15 minutes |
| **Troubleshooting** | Problem diagnosis and resolution | 400-1000 words | 10-30 minutes |
| **Reference** | Comprehensive information resource | 500-2000 words | N/A (reference) |
| **Policy** | Official organizational policies | 300-1500 words | N/A (awareness) |

### 3.2 FAQ Articles

**Purpose**: Provide quick, concise answers to frequently asked questions.

**Structure**:
```
1. Question (clear, searchable)
2. Short Answer (1-2 sentences)
3. Detailed Explanation (if needed)
4. Related FAQs
5. Last Updated
```

**Characteristics**:
- Single question per article
- Answer first, explanation second
- Use conversational tone
- Include examples where helpful

**Template**:
```markdown
# [Question as Title]

## Quick Answer
[1-2 sentence direct answer]

## Detailed Explanation
[Expanded explanation with context]

## Example
[Practical example if applicable]

## Related FAQs
- [Link to related FAQ 1]
- [Link to related FAQ 2]

---
Last Updated: [Date] | Article ID: FAQ-XXX
```

### 3.3 How-To Articles

**Purpose**: Guide users through specific tasks or procedures.

**Structure**:
```
1. Title (action-oriented)
2. Summary (what user will accomplish)
3. Prerequisites (what's needed before starting)
4. Step-by-Step Instructions (numbered)
5. Verification (how to confirm success)
6. Troubleshooting (common issues)
7. Related Articles
```

**Characteristics**:
- Action-oriented titles ("How to...", "Setting up...")
- Numbered steps
- One action per step
- Screenshots for UI actions
- Expected results after each major step

**Template**:
```markdown
# How to [Perform Action]

## Summary
[Brief description of what this guide accomplishes]

**Estimated Time**: X minutes  
**Difficulty**: Beginner/Intermediate/Advanced  
**Prerequisites**: [List requirements]

- [Item 2]

## Instructions

### Step 1: [Action Description]
[Detailed instructions]

[Screenshot if applicable]

### Step 2: [Action Description]
...

## Verification
[How to confirm the task was completed successfully]

## Troubleshooting
| Issue | Solution |
|-------|----------|
| [Problem] | [Solution] |

## Related Articles
- [Link to related article 1]

---
Last Updated: [Date] | Article ID: HOW-XXX
```

### 3.4 Troubleshooting Articles

**Purpose**: Help users diagnose and resolve problems.

**Structure**:
```
1. Title (problem-based)
2. Symptoms (how to identify the issue)
3. Possible Causes
4. Diagnostic Steps
5. Solutions (organized by cause)
6. Prevention (how to avoid future occurrences)
7. When to Escalate
```

**Characteristics**:
- Symptom-based organization
- Decision-tree format
- Multiple solution paths
- Clear escalation triggers

**Template**:
```markdown
# Troubleshooting: [Problem Description]

## Symptoms
- [Symptom 1]
- [Symptom 2]

## Quick Checks
1. [Immediate check 1]
2. [Immediate check 2]

## Solutions

### Solution A: [Cause Description]
**When to use**: [Symptoms indicating this solution]

1. [Step 1]
2. [Step 2]

...

## Prevention
[Steps to prevent recurrence]

## Escalation

3. Include [required details]

---
Last Updated: [Date] | Article ID: TRO-XXX
```

## 5. Writing Guidelines

### 5.1 Style Guide

#### 5.1.1 Voice and Tone

| Attribute | Description | Example |
|-----------|-------------|---------|
| **Clear** | Use simple, direct language | "Click the Save button" not "Utilize the Save functionality" |
| **Helpful** | Focus on user success | "You'll see a confirmation message" |
| **Professional** | Maintain appropriate formality | Avoid slang, emojis (except in customer-facing) |
| **Empathetic** | Acknowledge user frustration | "If you're experiencing this issue..." |
| **Action-Oriented** | Lead with verbs | "Download the file" not "The file should be downloaded" |

#### 5.1.2 Language Standards

- **Active Voice**: Use active voice whenever possible
  - âœ“ Click the Submit button
  - âœ— The Submit button should be clicked

- **Second Person**: Address the reader directly
  - âœ“ You will see a confirmation
  - âœ— The user will see a confirmation

- **Present Tense**: Use present tense for instructions
  - âœ“ The page displays
  - âœ— The page will display

- **Parallel Structure**: Use consistent grammatical structure
  - âœ“ Save, print, and close
  - âœ— Save, printing, close

#### 5.1.3 Inclusive Language

| Avoid | Use Instead |
|-------|-------------|
| He/She | They, you, or the user |
| Dummy data | Sample data, test data |
| Master/Slave | Primary/Secondary, Leader/Follower |

## 8. Search and Discovery

### 8.1 Keywords

#### 8.1.1 Keyword Strategy

- Include synonyms and alternate terms
- Use words users actually search for
- Include common misspellings (if platform supports)
- Add Bengali equivalents for local terms
- Include product names and model numbers

#### 8.1.2 Keyword Placement

| Location | Purpose | Best Practice |
|----------|---------|---------------|
| Title | Primary ranking factor | Include primary keyword |
| Summary | Secondary ranking | Include 2-3 keywords |
| Body | Context relevance | Natural usage |
| Tags | Faceted search | Controlled vocabulary |
| Alt text | Image search | Descriptive keywords |

### 8.2 Search Optimization

#### 8.2.1 Title Optimization

- Front-load important keywords
- Use natural language
- Include action words for how-to articles
- Keep under 60 characters

| Poor | Improved |
|------|----------|
| Password | How to Reset Your Password |
| ERP Issue | Troubleshooting ERP Login Issues |
| Farm | Farm Management: Animal Registration Guide |

#### 8.2.2 Content Optimization

- Use headings that match search queries
- Include FAQ schema markup (if supported)
- Create comprehensive content that answers follow-up questions
- Link related articles to improve discovery

### 8.3 Related Content

#### 8.3.1 Linking Strategy

- Link to prerequisite articles
- Link to next-step articles
- Link to related troubleshooting
- Link to policy documents
- Maximum 5-7 related links per article

#### 8.3.2 "You Might Also Need" Suggestions

Contextually suggest content based on:
- Current article category
- User role
- Article type
- Popular next articles (analytics-driven)

# Troubleshooting: Cannot Connect to ERP System

## Symptoms
- Browser shows "This site can't be reached" or "Connection timeout"
- Login page loads but credentials don't work
- "Server Error 500" or "Service Unavailable" messages
- ERP page loads partially with missing elements
- "Session expired" errors immediately after login

## Quick Checks
Before proceeding with detailed solutions, verify:
1. [ ] Other websites load correctly (test: google.com)
2. [ ] Colleagues can access the ERP (ask in Teams)
3. [ ] You're using the correct URL: https://erp.smartdairy-bd.com
4. [ ] Your internet connection is active

## Solutions

### Solution A: Network Connectivity Issue
**Indications:**
- No websites load
- "No internet" icon in system tray
- Other network-dependent apps not working

**Steps:**
1. Check WiFi/Ethernet connection
2. Restart your router (unplug 30 seconds)
3. Try connecting to mobile hotspot
4. Contact IT if office network is down

### Solution B: Browser Cache Issue
**Indications:**
- ERP worked previously but now doesn't
- Page loads partially or looks broken
- Error clears in incognito/private mode

**Steps (Chrome):**
1. Press <kbd>Ctrl</kbd> + <kbd>Shift</kbd> + <kbd>Delete</kbd>
2. Select "Cached images and files"
3. Click "Clear data"
4. Close and reopen browser
5. Try ERP again

### Solution C: VPN Required
**Indications:**
- Error only occurs when working remotely
- Works fine in office

**Steps:**
1. Connect to Smart Dairy VPN first
2. Open VPN client (Cisco AnyConnect)
3. Enter credentials and connect
4. Once connected, try ERP again

### Solution D: System Maintenance
**Indications:**
- "Service Unavailable" or "Maintenance in progress"
- Multiple users experiencing same issue

**Action:**
Check system status at: https://status.smartdairy-bd.com

## Prevention
- Keep browser updated to latest version
- Clear cache monthly
- Use bookmarks to ensure correct URL
- Save important work before scheduled maintenance windows

## Escalation
If none of the above solutions resolve the issue:

1. Collect the following information:
   - Screenshot of error message
   - Time issue started
   - Your location (office/remote)
   - Browser name and version

2. Submit ticket to: https://smartdairy.atlassian.net/servicedesk
   - Priority: P2 (if working remotely), P3 (if in office)
   - Category: Applications > ERP

3. For urgent issues affecting critical work:
   - Call IT Help Desk: ext. 8888
   - WhatsApp (urgent): +880-XXX-XXXXXX
