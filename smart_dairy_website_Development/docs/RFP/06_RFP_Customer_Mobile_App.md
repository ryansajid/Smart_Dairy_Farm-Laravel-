## 1. Executive Summary

### 1.1 Project Overview

Smart Dairy Ltd., a pioneering organic dairy farming enterprise in Bangladesh and subsidiary of the renowned Smart Group, seeks to develop a comprehensive Business-to-Consumer (B2C) mobile application. This application will serve as the primary digital channel for customers to discover, purchase, and receive Smart Dairy's premium organic dairy products directly at their doorsteps.

The Smart Dairy Customer Mobile App represents a strategic initiative to digitize the company's sales channels, enhance customer engagement, and establish a direct-to-consumer (D2C) relationship that bypasses traditional retail intermediaries. This mobile-first approach aligns with Bangladesh's rapidly growing smartphone penetration (currently exceeding 100 million users) and the accelerated adoption of e-commerce following the digital transformation triggered by recent global events.

### 1.2 Business Objectives

The primary objectives of this mobile application are:

| Objective | Target KPI | Timeline |
|-----------|------------|----------|
| Customer Acquisition | 50,000 app downloads in Year 1 | 12 months post-launch |
| Revenue Generation | BDT 50 million in app-based sales (Year 1) | 12 months post-launch |
| Subscription Conversion | 30% of active users on subscription plans | 6 months post-launch |
| Customer Retention | 70% monthly retention rate | Ongoing |
| Order Frequency | Average 4 orders per customer per month | 6 months post-launch |
| App Store Rating | Minimum 4.5 stars on both platforms | Ongoing |

### 1.3 Target Market

**Primary Target Audience:**
- Urban and suburban households in Dhaka, Chattogram, and major divisional cities
- Health-conscious consumers aged 25-55
- Middle to upper-middle-class families with monthly income above BDT 50,000
- Parents prioritizing organic, chemical-free food for children
- Working professionals seeking convenient daily essentials delivery

**Secondary Target Audience:**
- Restaurants and cafes seeking premium organic dairy suppliers
- Fitness enthusiasts and athletes requiring high-quality protein sources
- Elderly consumers prioritizing digestive health and nutrition
- Environmentally conscious consumers supporting sustainable farming

### 1.4 Product Categories

The mobile application will support the following Smart Dairy product portfolio:

| Category | Products | Delivery Type |
|----------|----------|---------------|
| Fresh Milk | Saffron Organic Milk (1000ml), Saffron Low-Fat Milk (1000ml) | Daily/Subscription |
| Fermented Products | Saffron Sweet Yogurt (500g), Saffron Plain Yogurt (500g), Saffron Mattha (1000ml) | Daily/On-demand |
| Dairy Delicacies | Saffron Ghee (500g), Saffron Butter (200g), Saffron Cream (250ml) | On-demand |
| Cheese Varieties | Saffron Paneer (500g), Saffron Cheddar (200g), Saffron Mozzarella (200g) | On-demand |
| Premium Products | Saffron A2 Milk (1000ml), Saffron Deshi Ghee (500g) | On-demand |
| Meat Products | Saffron Organic Beef (2kg pack) | Scheduled Delivery |
| Gift Packages | Saffron Premium Gift Box, Saffron Health Pack | On-demand |

### 1.5 Unique Value Propositions

The Smart Dairy Customer App will differentiate itself through:

1. **Farm-to-Home Freshness**: Direct sourcing from Smart Dairy'ss organic farm in Rupgonj, ensuring products reach customers within 24 hours of production

2. **Subscription Simplicity**: One-tap subscription management for daily essentials like milk, yogurt, and mattha with flexible pause/resume capabilities

3. **Quality Transparency**: Blockchain-enabled traceability showing product journey from cow to customer, including milking time, quality test results, and delivery chain

4. **Organic Certification**: Digital certificates and lab reports accessible for every product batch

5. **Smart Rewards Program**: Gamified loyalty system with dairy-themed achievements, referral bonuses, and exclusive member benefits

6. **Community Features**: Recipe sharing, nutrition tips, and direct communication with dairy experts

### 1.6 Project Scope

**In Scope:**
- Native iOS application (Swift/SwiftUI)
- Native Android application (Kotlin/Jetpack Compose)
- Customer-facing mobile app with complete e-commerce functionality
- Subscription management system integration
- Real-time order tracking with GPS integration
- Payment gateway integration (bKash, Nagad, cards, COD)
- Push notification system
- In-app customer support chat
- Review and rating system
- Offline browsing capability
- Dark mode support
- Multi-language support (English, Bangla)

**Out of Scope:**
- Admin dashboard (separate RFP)
- Delivery driver application (separate RFP)
- Inventory management system (existing system integration)
- ERP integration (phase 2)

### 1.7 Success Criteria

The project will be deemed successful upon meeting the following criteria:

1. **Functional Completeness**: All features specified in Section 4 implemented and tested
2. **Performance Standards**: App launch time under 3 seconds, API response time under 500ms
3. **Uptime SLA**: 99.9% availability during business hours (6 AM - 10 PM)
4. **Security Compliance**: Passing security audit with no critical vulnerabilities
5. **User Acceptance**: Beta testing completion with NPS score above 50
6. **App Store Approval**: Successful publication on both Apple App Store and Google Play Store
7. **Load Testing**: Supporting 10,000 concurrent users without performance degradation

## Extended Technical Specifications

### Database Schema Design

#### User Management Schema

```sql
-- Users Table
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    phone VARCHAR(15) UNIQUE NOT NULL,
    email VARCHAR(255) UNIQUE,
    password_hash VARCHAR(255),
    first_name VARCHAR(50),
    last_name VARCHAR(50),
    date_of_birth DATE,
    gender VARCHAR(10),
    avatar_url TEXT,
    preferred_language VARCHAR(10) DEFAULT 'en',
    status VARCHAR(20) DEFAULT 'ACTIVE',
    email_verified BOOLEAN DEFAULT FALSE,
    phone_verified BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    last_login_at TIMESTAMP WITH TIME ZONE,
    metadata JSONB DEFAULT '{}'::jsonb
);

-- User Addresses Table
CREATE TABLE user_addresses (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    label VARCHAR(20) NOT NULL, -- HOME, WORK, OTHER
    is_default BOOLEAN DEFAULT FALSE,
    recipient_name VARCHAR(100),
    phone VARCHAR(15),
    address_line_1 VARCHAR(255) NOT NULL,
    address_line_2 VARCHAR(255),
    area VARCHAR(100) NOT NULL,
    city VARCHAR(50) NOT NULL,
    postcode VARCHAR(10),
    instructions TEXT,
    latitude DECIMAL(10, 8),
    longitude DECIMAL(11, 8),
    is_serviceable BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- User Devices Table (for push notifications)
CREATE TABLE user_devices (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    device_id VARCHAR(255) NOT NULL,
    device_type VARCHAR(10) NOT NULL, -- ios, android
    device_token TEXT NOT NULL,
    app_version VARCHAR(20),
    os_version VARCHAR(20),
    model VARCHAR(50),
    is_active BOOLEAN DEFAULT TRUE,
    last_used_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    UNIQUE(user_id, device_id)
);

-- User Preferences Table
CREATE TABLE user_preferences (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL UNIQUE REFERENCES users(id) ON DELETE CASCADE,
    notification_email BOOLEAN DEFAULT TRUE,
    notification_sms BOOLEAN DEFAULT TRUE,
    notification_push BOOLEAN DEFAULT TRUE,
    notification_promotions BOOLEAN DEFAULT TRUE,
    dark_mode VARCHAR(10) DEFAULT 'system', -- light, dark, system
    quiet_hours_start TIME,
    quiet_hours_end TIME,
    preferred_delivery_time VARCHAR(20),
    dietary_restrictions JSONB DEFAULT '[]'::jsonb,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Create indexes for performance
CREATE INDEX idx_users_phone ON users(phone);
CREATE INDEX idx_users_email ON users(email) WHERE email IS NOT NULL;
CREATE INDEX idx_user_addresses_user_id ON user_addresses(user_id);
CREATE INDEX idx_user_addresses_default ON user_addresses(user_id, is_default) WHERE is_default = TRUE;
CREATE INDEX idx_user_devices_user_id ON user_devices(user_id);
CREATE INDEX idx_user_devices_token ON user_devices(device_token);
```

```sql
    track_inventory BOOLEAN DEFAULT TRUE,

    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),

    created_by UUID REFERENCES users(id),
    updated_by UUID REFERENCES users(id)
);

-- Product Attributes Table

    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    attribute_value VARCHAR(255) NOT NULL,

    display_order INTEGER DEFAULT 0
);
```

```sql
CREATE TABLE orders (
```

```sql
-- Subscriptions Table
CREATE TABLE subscriptions (

    time_slot VARCHAR(20) NOT NULL,
    
    -- Duration
    start_date DATE NOT NULL,
    end_date DATE, -- NULL for ongoing
    duration_type VARCHAR(20) DEFAULT 'ONGOING', -- ONGOING, FIXED
    
    -- Delivery
    delivery_address_id UUID NOT NULL REFERENCES user_addresses(id),
    payment_method_id UUID, -- Reference to saved payment method
    
    -- Pause Info
    paused_at TIMESTAMP WITH TIME ZONE,
    pause_reason VARCHAR(50),
    resume_date DATE,
    
    -- Cancellation
    cancelled_at TIMESTAMP WITH TIME ZONE,
    cancellation_reason TEXT,
    
    -- Metadata
    total_deliveries INTEGER DEFAULT 0,
    successful_deliveries INTEGER DEFAULT 0,
    total_savings DECIMAL(10, 2) DEFAULT 0,
    
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Subscription Deliveries Table
CREATE TABLE subscription_deliveries (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    subscription_id UUID NOT NULL REFERENCES subscriptions(id) ON DELETE CASCADE,
    order_id UUID REFERENCES orders(id),
    
    scheduled_date DATE NOT NULL,
    status VARCHAR(20) DEFAULT 'SCHEDULED', -- SCHEDULED, PROCESSING, DELIVERED, SKIPPED, CANCELLED
    
    -- Modifications
    modified_quantity INTEGER,
    modified_product_id UUID REFERENCES products(id),
    modification_reason TEXT,
    
    -- Skip Info
    is_skipped BOOLEAN DEFAULT FALSE,
    skip_reason VARCHAR(50),
    
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Indexes
CREATE INDEX idx_subscriptions_user_id ON subscriptions(user_id);
CREATE INDEX idx_subscriptions_status ON subscriptions(status);
CREATE INDEX idx_subscription_deliveries_subscription ON subscription_deliveries(subscription_id);
CREATE INDEX idx_subscription_deliveries_date ON subscription_deliveries(scheduled_date);
```

### Key Metrics Dashboard

| Metric | Formula | Target | Frequency |
|--------|---------|--------|-----------|
| Daily Active Users (DAU) | Count of unique users with session | 8,000 | Daily |
| Monthly Active Users (MAU) | Count of unique users with session in month | 25,000 | Monthly |
| DAU/MAU Ratio | DAU / MAU | 30% | Daily |
| Session Duration | Average time per session | 4 minutes | Daily |
| Screens per Session | Average screens viewed | 8 | Daily |
| Retention (Day 1) | Users returning next day / New users | 40% | Daily |
| Retention (Day 7) | Users returning day 7 / New users | 25% | Weekly |
| Retention (Day 30) | Users returning day 30 / New users | 15% | Monthly |
| Cart Abandonment Rate | Carts abandoned / Carts created | < 60% | Daily |
| Checkout Conversion | Orders placed / Checkout started | > 70% | Daily |
| Average Order Value | Total revenue / Number of orders | BDT 800 | Daily |
| Subscription Conversion | Subscribers / Active users | 30% | Monthly |
